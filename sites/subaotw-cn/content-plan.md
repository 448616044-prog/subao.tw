## 🔄 内容新鲜度节奏

Baiduspider 对新站的核心信号之一：**内容多久更新一次**。
当前状态：全部页面 7/3 批量更新，无自然节奏 → 百度判定为「一次性批量建站」，收录意愿低。

### 方案：每周自然更新 3 页

每周选 3 个不同目录的页面，做微小更新（改 lastmod + 补充 1-2 句话 + 更新 `upDate`），
模拟「真实运营中的网站」。

| 周次 | 更新页面 | 改动内容 |
|------|---------|---------|
| W1 | index.html | 补充最新案例/客户数 |
| W1 | tw-to-cn/food-shipping.html | 补充时令食品提示 |
| W1 | guide/shipping-methods.html | 补充运输方式对比 |
| W2 | equipment/cnc-export-taiwan.html | 补充最新型号案例 |
| W2 | blog/electronics-shipping.html | 补充电池运输新规 |
| W2 | tw-to-cn/cosmetics-shipping.html | 补充夏季化妆品运输提示 |
| W3 | index.html | 更新服务数据 |
| W3 | faq.html | 新增2条FAQ |
| W3 | cases/case-cnc-dongguan.html | 补充细节参数 |

### 规则
- 每次更新修改 `lastmod` 和 `upDate` 为当天日期
- 每次至少改动 50 字以上（百度判定不是刷更新）
- 每周日执行一次，3页一起推

## 💬 百度知道自问自答库（10条）

百度知道对搜索用户的Q&A类查询有极高权重，且自身页面在百度有天然排名优势。

### 操作方式
- 用2个不同百度账号：一个提问、一个回答
- 回答中自然植入「速豹集运」品牌（但不要硬广）
- 提问用真实用户会搜索的问法
- 每天只发1-2条，避免被判定为营销号

| # | 提问 | 回答要点 | 植入方式 |
