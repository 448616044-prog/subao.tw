## 三、UI 升级方案（重大改造 / 3 阶段）

### 阶段 1：导航 + Footer 统一（最优先 / 1-2 天）

**目标**：全站 130 个页面导航和 footer 视觉一致

**做法**：
1. **Header 标准化**（参考 index.html 的 8 项导航）：
   ```
   台灣發大陸 → 運費 (dropdown: 報價/試算) → 可以寄嗎 → 關稅速查 → 材積計算 → 文章攻略 → 更多 (dropdown: 上門取貨/常見問題/快遞對比/敏感貨攻略/2026報告/退件Top10/合作夥伴/關於我們) + LINE 咨詢按钮
   ```

2. **Footer 标准化**（4 列，与 index.html 一致）：
   - 列 1：速豹集運 + LINE
   - 列 2：服務項目（8 项：上門取貨/台灣發大陸/運費報價/運費試算/關稅速查/寄件攻略/常見問題/合作夥伴）
   - 列 3：**敏感貨專區**（4 项：保健品/茶葉/化妝品/食品）
   - 列 4：聯系我們（LINE + Email + 關於）

3. **执行方式**：
   - 写一个 Python 脚本批量替换 130 个 HTML 文件的 `<header>...</header>` 和 `<footer>...</footer>` 块
   - 同步替换对应的 `<style>` 块中的 `.header`、`.footer` CSS
   - 注意：必须用**完全相同的模板**替换，参考已经修好的 index.html + pricing.html + pickup-service.html

**风险评估**：
- ⚠️ 上次失败过（`git reset --hard d45f94f`）—— 那次的失败是因为只替换 HTML 没同步替换 CSS
- 这次必须**先在 3 个测试页**（已修好的 index/pricing/pickup-service）验证 CSS 兼容性，再批量推
- 备份点已就绪（tag `backup-2026-07-02-pre-ui-redesign`）

---

### 阶段 2：CSS 变量化 + 主题切换（1-2 周）

**目标**：把分散在 130 个页面里的 CSS 抽到统一变量系统，支持未来主题切换

**做法**：
1. 定义全局 CSS 变量（在每个页面 `<style>` 顶部）：
   ```css
   :root {
     /* 主色 */
     --primary: #0066CC;
     --primary-dark: #004C99;
     --primary-light: #E6F0FA;
     /* 辅色 */
     --secondary: #E65100;
     --line-green: #008A00;
     /* 中性色 */
     --bg-white: #FFFFFF;
     --bg-gray: #F5F7FA;
     --bg-dark: #1A1A2E;
     --text-dark: #333333;
     --text-light: #666666;
     /* 阴影 / 圆角 / 过渡 */
     --shadow: 0 2px 8px rgba(0,0,0,0.1);
     --radius: 8px;
     --transition: all 0.3s ease;
   }
   ```

2. 用 **CSS Audit 工具** 扫出各页面里 `color: #...` 的硬编码色值，逐一替换为变量
3. 后续可加暗色模式（仅切换变量值）

**预期收益**：
- 改色一次，全站生效
- 便于品牌升级（万一改主色，全站不用手动改 100+ 文件）

---

### 阶段 3：组件库 + 设计系统（1-2 月）

**目标**：抽取公共组件，所有页面从"各自实现"变为"调组件"

**关键组件**：
- `Header` (8 项桌面 + 移动端 hamburger)
- `Footer` (4 列)
- `CTA Section` (统一 CTA 区块)
- `Card` (通用卡片)
- `Button` (primary/secondary/line 3 种)
- `Table` (带 sort + responsive)
- `FAQ` (accordion)
- `Breadcrumb` (与 schema 对应)

**实施方式**（静态站无构建工具）：
- 写一个 Python 脚本 `build.py`，把每个页面当作"模板字符串"，用 `{{component}}` 占位
- 改组件时只改 Python 脚本，重新生成所有页面
- 130 个页面 × 8 个组件 = 1040 处复用点，改一次全站统一

**风险**：
- 静态站引入"伪模板"会破坏"每个 HTML 自包含 CSS"的现有约定
- 必须保留 `git` 单文件可回滚
- 可能与 GSC 已索引的页面 hash 冲突（影响 SEO 排名）

## 四、UI 长期方向（半年到 1 年）

### 4.1 视觉品牌升级
- **Logo 重设计**：当前 webp/png 双格式，缺少 SVG 高保真版
- **品牌色微调**：从 `#0066CC` 考虑改为更稳重的 `#003366`（深蓝）
- **字体系统**：Noto Sans TC → 引入 `Inter` + `Noto Serif TC` 双字体（标题用 Serif）

### 4.2 信息架构升级
- **顶部 Mega Menu**：把"更多"下拉变成 Mega Menu 8 大分类
- **底部 sitemap**：footer 加 4 链接
- **面包屑导航**：每页都加（已有 schema，UI 还没全加）

### 4.3 转化路径优化
- **A/B Test 框架**：用 GA4 + Cloudflare Workers 跑实验
- **多步骤 LINE 表单**：把单按钮换成"分步表单"提高留存
- **Trust Signal 强化**：6年+ 2000+ 客户数 + LINE 截图 滚动展示

### 4.4 内容侧增强
- **AI 自动问答**：用 Cloudflare AI Gateway 接 OpenAI，提供"输入问题→AI 答"组件
- **运费试算 v2**：从"输入重量+目的地"升级为"上传照片→AI 识别品类+估算"
- **客户案例库**：每 2 周新增 1 个真实客户案例（需客户授权）
